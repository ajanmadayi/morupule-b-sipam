Notification to PTW Workflow

How to create a plant notification, convert it to a Work Request, issue a Work Order, control PTW / LoA / SFT, and close the record.

Maintenance PlannerPrepare Work Order plan, maintenance code, man-hours, permit requirement, supply and artisans.

System AdministratorCan perform all roles during commissioning, testing, and system support.

Step-by-Step Procedure

  1. Create Event / Notification

    Open Event Log, create a main entry, select the affected KKS asset, enter observation, informant, date/time, and keep state as Open.

    In this application, the operational notification starts as an Event Log entry.

  2. Create Work Request

    From the open Event detail, click Create work request. The KKS must match the Event KKS. Enter work name, type of work, department, CMPT details, and observation.

    Status after saving: submitted.

  3. Approve Work Request

    The Maintenance Approver reviews the request. If valid, click Approve request. S-PULSE creates the Work Order automatically.

    Status after approval: Work Request approved, Work Order planning.

  4. Plan Work Order

    The Maintenance Planner completes description of work, maintenance code, equipment condition, workplace requirements, expected man-hours, supply, artisans, and permit decision.

    Permit decision must be one of: No permit required, PTW, LoA, or SFT.

  5. Approve Plan

    The Maintenance Approver approves the plan. If PTW / LoA / SFT is required, the Work Order moves to permit decision and waits for a linked safety document.

    Status after approval: Work Order permit_decision, permit status required.

  6. Prepare PTW / LoA / SFT

    The Shift Leader opens PTW, LoA & SFT or uses the Work Order permit button. Select the correct form type, confirm KKS, enter location, issued-to person, isolations, precautions, and assessments.

    Status after saving: Permit prepared.

  7. Issue and Receive Permit

    Issue the permit after confirming precautions and controller name. The work party must then receive the permit before execution can start.

    Valid permit sequence: prepared to issued to received.

  8. Confirm Execution

    The Shift Leader confirms execution only after the linked permit is received. S-PULSE blocks execution if the required permit is missing or only issued.

  9. Complete Work

    The responsible technician enters completion summary, actual man-hours, failure mode/cause, downtime start, and restored time if applicable.

    Status after completion: Work Order work_check.

  10. Clear and Cancel Permit

    The Shift Leader clears the work location and cancels the permit after restoration. S-PULSE will not allow Work Order acceptance while the linked permit remains prepared, issued, received, or cleared.

    Final permit status must be cancelled.

  11. Accept and Close Work Order

    The Maintenance Approver reviews the completed work. If satisfactory, enter acceptance remarks and click Accept work.

    Final status: Work Order closed. The source Event / Notification closes automatically.

Status Rules

RecordStatus sequenceImportant rule
Event / Notificationopen to closedAuto-closes after linked Work Order is accepted.
Work Requestsubmitted to approved or declinedOnly open Events can originate a Work Request.
Work Orderplanning to plan_approval to permit_decision to execution to work_check to closedRequired planning fields must be completed before plan submission.
Permitprepared to issued to received to cleared to cancelledWork Order cannot close until linked permit is cancelled.
