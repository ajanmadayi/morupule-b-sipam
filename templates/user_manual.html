Login and Roles

  1. Sign in

    Open SIPAM and enter your employee number and password. User permissions are controlled by your assigned role.

  2. Role-based screens

    Some actions appear only for the correct role. Example: Shift Leader can prepare permits, Maintenance Planner can prepare work plans, Maintenance Approver can approve and accept work.

  3. Sign out

    Use the sign-out button in the top right when finished, especially on shared computers.

Event Log / Notification

  1. Create a main event

    Open Event Log, click New main entry, select logbook, KKS asset, date/time, observation, and informant.

  2. Add sub-entry

    Select a main event and click New sub-entry to add follow-up comments without creating another main notification.

  3. Create Work Request

    For maintenance action, open an open main event and click Create work request. The Work Request inherits the KKS relationship.

  4. Close event

    Events can be closed manually with a reason, or automatically when the linked Work Order is accepted and closed.

In this app, the word notification means the Event Log entry that starts the maintenance workflow.

KKS Asset Directory

  1. Search asset

    Open Asset Directory and search by KKS code, description, or responsible area.

  2. Review history

    Select a KKS to see linked Events, Corrective Maintenance, Preventive Maintenance, and Permits.

  3. Export KKS

    Click Export to download the full KKS directory. Search first if you want only filtered KKS records.

Corrective Maintenance

2Approve
3Plan Work Order
4Permit Decision
5Execution
6Acceptance

  1. Submitted Work Request

    New Work Requests appear as submitted. Maintenance Approver reviews and either approves or declines.

  2. Work Order planning

    Planner enters description of work, maintenance code, equipment condition, workplace requirements, expected man-hours, supply, artisans, and permit requirement.

  3. Permit decision

    Choose No permit required, PTW, LoA, or SFT. Required permits must be linked before execution.

  4. Complete work

    Technician records completion summary, actual man-hours, failure mode/cause, downtime start, and restored time where applicable.

  5. Accept work

    Maintenance Approver accepts the work after checking completion. Work Order status becomes closed.

PTW, LoA & SFT

  1. Prepare permit

    Open PTW, LoA & SFT or use the permit button from a Work Order. Select form type and complete KKS, location, issued-to, employer, isolations, precautions, and assessments.

  2. Issue permit

    Issue only after safety precautions are confirmed. Controller name and remarks are required.

  3. Receive permit

    Work party receives the permit before execution starts. Work Orders requiring a permit cannot enter execution until the permit is received.

  4. Clear and cancel

    After work is complete, clear the work location and cancel the permit. Work Order acceptance is blocked until linked permit is cancelled.

  5. Print permit

    Open a permit and click Print record for the formal PTW / LoA / SFT print format.

Preventive Maintenance

  1. Create schedule type

    Define interval, strategy, calendar unit, tolerance, and task type.

  2. Create recurrent task

    Select primary KKS, additional assets, department, duration, start date, and recurrence details.

  3. Generate calendar

    Use the preventive calendar to preview and generate monthly PM tasks.

  4. Complete PM task

    Open a generated task, enter actual man-hours and completion feedback.

Administration

  1. Users & Roles

    Create users, assign role, activate/deactivate accounts, and reset passwords.

  2. Logbooks

    Create and maintain operational logbooks and responsibility rules.

  3. Audit Trail

    Review sign-ins, created records, updates, approvals, and workflow actions.

  4. System Status

    Check database health, KKS import history, record counts, backups, and restore history.

  5. KKS Import

    Validate and commit KKS Excel workbooks. Only System Administrator can import KKS.
